试题二. 某电话公司决定开发一个管理所有客户信息的交互式网络系统。系统的功能如下：
1.	浏览客户信息：任何使用Internet的网络用户都可以浏览电话公司所有的客户信息(包括姓名、住址、电话号码等)。 
2.	登录：电话公司授予每个客户一个帐号。拥有授权帐号的客户，
        可以使用系统提供的页面设置个人密码，并使用该帐号和密码向系统注册。
3.	修改个人信息：客户向系统注册后，可以发送电子邮件或者使用系统提供的页面，对个人信息进行修改。
4.	删除客户信息：只有公司的管理人员才能删除不再接受公司服务的客户的信息。
问题：根据描述建立该系统的动态模型（事件跟踪图+状态图）和功能模型（用例图）。

编写事件脚本
正常：

  1. 客户信息表 授予 客户 账号
  2. 客户 使用 被授予的 账号 进行注册
  3. 系统 要求 客户 输入密码
  4. 用户输入密码
  5. 客户登录
  6. 客户信息类 显示 客户个人信息
  7. 用户修改客户信息
  8. 显示修改成功，客户信息类 更新 客户个人信息
  9. 用户退出，系统关闭。
  1. 管理者进入系统
  2. 系统显示所有客户信息
  3. 管理者要删除客户信息
  4. 管理者添加新客户信息，授予客户账号
  5. 管理员退出,系统关闭。
  1. 网络用户进入系统浏览客户信息
  2. 系统显示所有客户信息

异常:

  1. 客户进行登录
  2. 密码错误 登录失败
  3. 客户重新登录
  4. 登录成功,显示客户个人信息
  5. 客户退出,系统关闭
